# 多言語対応 (i18n) の方針

2026-07-20: `<html lang="en">`（Vite テンプレートの残骸）を `lang="ja"` に修正した際に
決めた、将来の多言語対応の設計方針。

## 現状

- UI 文字列はすべて日本語ハードコード。`lang="ja"` 固定
- privacy.html / terms.html は日本語 + 英語併記の単一ページ

## 対応するときの設計

1. **ロケール解決**: 起動時に `navigator.language`（+ 設定での明示切替）から
   ロケールを決め、`document.documentElement.lang` を書き換える。
   HTML の静的 lang はフォールバックとして "ja" のまま
2. **文字列**: `src/i18n/messages.<locale>.ts` にキー付きで抽出。
   ライブラリは最小（自前の `t()` か軽量な i18n ライブラリ）。
   まず ja を正とし en を追加する
3. **日時・曜日**: 手組みのフォーマッタ（gridMetrics の formatTime 等）を
   `Intl.DateTimeFormat(locale, ...)` / Temporal の `toLocaleString` ベースへ寄せる。
   週の開始曜日（月曜固定）もロケール/設定化の候補
4. **ポリシーページ**: /privacy.html は併記のままでよいが、量が増えたら
   /en/privacy.html に分離
5. **ブランド語彙**: 「枡」「吉日」の説明は翻訳せず、ローマ字 + 短い説明で扱う
   （プロダクト名の一部として）

## 注意

- Google 由来のデータ（予定タイトル等）はユーザーの言語のまま表示する
- 祝日カレンダー等はユーザーが選ぶものなので UI ロケールと独立
