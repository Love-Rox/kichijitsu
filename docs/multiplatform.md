# デスクトップ・モバイル展開の方針

2026-07-20 のユーザー相談で確定した進め方。

## 前提

ローカルファースト構成（正本=Google、各デバイス=IndexedDB レプリカ、サーバー=トークンと
同期状態のみ）のため、**デバイスが増えても同期設計は変わらない**。各デバイスが自分の
セッションで同じ API に繋ぐだけ。

## 進める順序

```
本番デプロイ → PWA化(小) → フェーズ5(書き戻し) → モバイルレイアウト
→ Tauri desktop → (ストア配布が必要になったら) Tauri 2 mobile
```

1. **PWA 化**: manifest（アイコンは brand/tile.svg から）+ Service Worker で
   アプリシェルをキャッシュ。データは IndexedDB に既にあるため、これだけで
   オフライン起動が完成。デスクトップ(Chrome/Edge)・Android のインストール、
   iOS のホーム画面追加までカバー
2. **モバイルレイアウト（シェルに関係なく必要な本体作業）**（ユーザー要望 2026-07-20）:
   狭い画面(~640px 未満)向けに**選択できる複数のモバイルビュー**を用意する。Notion Calendar に倣う:
   - **3日タイムライン（モバイル既定）**: 時刻グリッドを N 日(既定3、1/3 を選択可)だけ横に並べる。
     週グリッドを「表示日数 N 可変の N 日タイムライン」に一般化するのが鍵 — 7=週、3/1=モバイル。
     既存の DayColumn・描画エンジン・packColumns をそのまま流用でき、実装が一本化できる
   - **月グリッド + 日アジェンダ**: 月表示で日をタップ → その日の予定一覧
   - 狭幅ではこれらを切替。PC は従来の週(7日)+月
   タッチ: タップ=詳細/アジェンダ、長押し=作成・移動。RRULE 展開・ストアは共通
3. **Tauri 2 デスクトップ**: まずリモート URL 方式（本番サイトを読む薄いガワ）
   — OAuth/cookie/CORS の面倒が発生しない。その上でトレイ常駐・グローバル
   ショートカット・ネイティブ通知を足す。フロントエンド同梱（オフラインバイナリ）は
   CORS + 認証方式の変更が必要なので後続
4. **Tauri 2 mobile**: ストア配布が必要になったら。同一コードベース。
   React Native 等への書き直しは仮想化レンダラーを捨てることになるので不採用

## 通知

予定リマインダーは Web Push (VAPID、Worker から送信)。iOS はインストール済み
PWA なら 16.4+ で対応。Tauri はネイティブ通知に差し替え。フェーズ5以降。
